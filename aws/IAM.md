# AWS Identity and Access Management (IAM)

## Overview
IAM is used to securely manage permissions for AWS services used in this project.

## Roles Used
- EC2 Instance Role
- ECR Access Role
- CloudWatch Logging Role
- Systems Manager Role

## Principle
- Least privilege access model
- Role-based access instead of user-based authentication

## Security Benefits
- No hardcoded credentials
- Temporary role-based authentication via STS
